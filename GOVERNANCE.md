# Governance

## Principles

1. **Open Governance** — All decisions transparent and public
2. **Axiom 0 Supremacy** — No decision may weaken Axiom 0
3. **Consensus-Seeking** — Discussion and consensus; voting is last resort

## Immutable Constraints

The following cannot be changed through any governance process:

- **Axiom 0**: Human Sovereignty and Wellbeing
- **No paid ranking**: Rankings must always be quality-based
- **Data ownership**: Bot data always belongs to Bot operator
- **Decentralization**: No single Yellow Page may be given exclusive authority
- **Open standard**: Anyone can build an AILP-compliant Yellow Page

## Decision-Making

### Specification Changes
1. Proposal via GitHub Issue (`spec-change` label)
2. 14-day discussion for non-trivial changes
3. ADR for architectural decisions
4. Axiom 0 compliance review
5. 2 maintainer approvals required

### Taxonomy Changes
- New categories: minor version bump, community ADR proposal
- Existing categories: never removed, only deprecated

## Roles

| Role | Responsibilities |
|------|-----------------|
| Maintainer | Review PRs, enforce Axiom 0, manage releases |
| Contributor | Submit PRs, participate in discussions |

## Versioning

AILP follows [Semantic Versioning](https://semver.org/):
- Major: breaking changes
- Minor: new skill categories, optional fields
- Patch: clarifications, fixes
